# concepts

Hier liegen optionale Feature-Konzepte vor der Umsetzung.

Ein Konzept ist sinnvoll, wenn ein Feature mehrere Module betrifft, fachliche Regeln konkretisiert oder vor dem Code eine Design-Entscheidung braucht.

## Format

```markdown
# Konzept - NW-### Feature-Name

## Ziel

## Akzeptanzkriterien

## Betroffene Module

## Fachliche Regeln

## Offene Fragen

## Umsetzungsplan
```

Kleine, eindeutige Features brauchen kein eigenes Konzept.
