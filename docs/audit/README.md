# audit

Hier liegen Codebase-, Security- und Architektur-Audits, sobald Code existiert.

Audits sind Momentaufnahmen. Sie dokumentieren Risiken, Inkonsistenzen, technische Schulden und empfohlene naechste Schritte.

## Namenskonvention

```text
YYYY-MM-DD-codebase.md
YYYY-MM-DD-security.md
YYYY-MM-DD-architecture.md
```

## Format

```markdown
# Audit - Thema - YYYY-MM-DD

## Scope

## Befunde

## Risiken

## Empfohlene Massnahmen

## Nachverfolgung
```

Gefundene Aufgaben werden bei Bedarf in `../backlog.md` oder `../decisions.md` referenziert.
